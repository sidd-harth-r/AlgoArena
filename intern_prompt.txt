What You Are BuildingAlgoArena is a web-based competitive programming platform — think LeetCode, but with one feature that does not exist anywhere else: after you submit code, the platform infers the Big-O time complexity of your solution using static AST analysis (not runtime benchmarking) and passes that complexity gap to an AI that generates a Socratic hint. The hint does not tell you the answer. It asks you the question that leads you there.Why it mattersEvery interviewer at Google, Meta, or Amazon will ask you to optimise your solution. LeetCode shows you pass/fail. AlgoArena shows you WHY your solution is slow and nudges you toward the insight — without spoiling it.As a portfolio piece, this project hits: systems design, algorithm analysis, AI integration, and production engineering. That is exactly what L4/L5 interviewers want to see.0.1 The Three Layers of FeedbackLayer 1 — Execution feedback (every judge has this): Pass / fail per test case, runtime in ms, error type (TLE, MLE, compile error, runtime error).Layer 2 — Complexity feedback (our differentiator): Static Big-O inference from the submitted AST. No other public judge does this.Layer 3 — Socratic AI hint (our differentiator): Complexity gap fed to Claude. The model is constrained to ask a question, not give the answer.0.2 What You Are NOT BuildingScope creep is how interns fail. The following are explicitly out of scope for this internship. Read the entire section before writing any code.PhilosophyWe are inferring Big-O from code structure, not measuring runtime. This is static analysis. An AST is a tree where each node represents a syntactic construct: a function definition, a for loop, a function call, etc. You walk this tree to detect patterns.pip install tree-sitter tree-sitter-pythonpython3 -c "from tree_sitter import Language, Parserimport tree_sitter_python as tspythonPY_LANGUAGE = Language(tspython.language())parser = Parser(PY_LANGUAGE)tree = parser.parse(b'for i in range(n): pass')print(tree.root_node.sexp())"4.2 The Inference RulesThese are the exact rules your engine must implement. They are applied in order of precedence (highest to lowest):PriorityPattern DetectedInferred Complexity1 (highest)A loop nested 3+ levels deep (for inside for inside for)O(n^3)2A recursive function with 2 recursive calls (no @lru_cache)O(2^n)3A loop nested exactly 2 levels deep (for inside for)O(n^2)4A call to sorted() or .sort() inside no loopO(n log n)5A single for or while loop (no nesting, no recursion)O(n)6A recursive function with @lru_cache decoratorO(n)7 (fallback)No loops, no recursion detectedO(1)4.3 _max_loop_depth(root, 0)    has_recursion, has_memo, branch_count = _check_recursion(root)    has_sort = _has_sort_call(root)    if max_loop_depth >= 3:                                    return Common Mistakes — Read Before You StartThese are real mistakes previous interns madeEach one cost at least 2 days of debugging. Read them now, not when you are stuck.Mistake 1: Blocking the FastAPI event loopIf you call Judge0 synchronously inside an async function using requests.get() instead of httpx.AsyncClient, every submission will block the entire server. Use async/await everywhere inside async def functions.Mistake 2: Storing secrets in gitYour .env and .env.local files must be in .gitignore before the first commit. Run: echo '.env' >> .gitignore && echo '.env.local' >> .gitignore before git add.Mistake 3: Polling without a timeoutYour frontend polling loop must have a maximum retry count (e.g., 20 retries x 1.5s = 30s timeout). Without it, a broken submission will poll forever and drain the user's battery.Mistake 4: Tree-sitter grammar version mismatchtree-sitter and tree-sitter-python must be compatible versions. Pin them: tree-sitter==0.21.3 and tree-sitter-python==0.21.0 in requirements.txt. Mixing major versions breaks the Language() constructor.Mistake 5: SSE connection not closed on component unmountIn AiTutorPanel, the AbortController ref must call abort() in a useEffect cleanup function. Without it, navigating away from a problem page while a hint is streaming will cause React state updates on unmounted components.useEffect(() => {  return () => { abortRef.current?.abort(); };   // cleanup on unmount}, []);Mistake 6: Test case format inconsistencyYour seed script must produce test case inputs in exactly the format the submitted code expects. For Two Sum, the input is two lines: the array on line 1, the target on line 2. Document this format in a comment next to each problem's test cases.Mistake 7: Modifying the AI prompt without approvalThe Socratic constraint in the system prompt is the product's intellectual contribution. If you modify it to 'be more helpful' by giving more direct answers, you break the product. Do not change the prompt without discussing it first.13.
